# Типи даних JavaScript

Всі мови програмування містять вбудовані типи даних, але часто відрізняються один від одного в різних мовах. Ця стаття – спроба описати вбудовані структури (типи) даних, доступні в JavaScript, та їх властивості. На їх основі будуються інші структури даних. Коли це можливо, ми будемо порівнювати типи даних у різних мовах.


## Динамічна типізація

JavaScript є слабо типізованою або динамічною мовою. Це означає, що вам не потрібно визначати тип змінної заздалегідь. Тип автоматично визначиться під час виконання програми. Також це означає, що ви можете використовувати одну змінну для зберігання різних типів даних:

<CodeEdit text={`
var foo = 42; // зараз foo типу Number
foo = "bar"; // а тепер foo типу String
foo = true;  // foo стає типу Boolean
`}/>

## Типы данных

Стандарт ECMAScript определяет 8 типов:

6  типів даних, що є примітивами:
* [Undefined](https://developer.mozilla.org/ru/docs/Glossary/undefined) (Невизначений тип) : ```typeof instance === "undefined"```
* [Boolean](https://developer.mozilla.org/ru/docs/Glossary/Boolean) (Логічний тип) : ```typeof instance === "boolean"```
* [Number](https://developer.mozilla.org/ru/docs/Glossary/Number) (Число) : ```typeof instance === "number"```
* [String](https://developer.mozilla.org/ru/docs/Glossary/String) (Рядок) : ```typeof instance === "string"```
* [BigInt](https://developer.mozilla.org/ru/docs/Glossary/BigInt) : ```typeof instance === "bigint"```
* [Symbol](https://developer.mozilla.org/ru/docs/conflicting/Web/JavaScript/Reference/Global_Objects/Symbol) (в ECMAScript 6) : ```typeof instance === "symbol"```
* [Null](https://developer.mozilla.org/ru/docs/Glossary/Null) (Null тип ) : ```typeof instance === "object"```. Спеціальний примітив, що використовується не тільки для даних але і як покажчик на фінальну точку в Ланцюжку Прототипов;
* [Object](https://developer.mozilla.org/ru/docs/Web/JavaScript/Reference/Global_Objects/Object) (Обєкт) : ```typeof instance === "object"```. Проста структура, яка використовується не тільки для зберігання даних, але й для створення інших структур, де будь-яка структура створюється з використанням ключового слова new: ```new object```, ```new array```, ```new map``` , ```new set```, ```new weakmap```, ```new weakset```, ```new Date``` та безліч інших структур;

І тут нам необхідно зробити застереження щодо використання оператора ```typeof``` для визначення типу структур,т.к. Всі структури будуть повертати "об'єкт" при його використанні, тому що призначення ```typeof``` - перевірка типу даних, але не структур. Якщо перевірити тип структури все ж таки необхідно, то в цьому випадку бажано використовувати оператор ```instanceof```, оскільки саме він відповідає на питання про те, який конструктор був використаний для створення структури.

Варто відзначити два особливі випадки роботи оператора ```typeof```: повернення ```"object"``` для значення ```null``` і ```"function"``` для функцій: перше прийнято вважати помилкою мови, збереженої для зворотньої сумісності, друге є умовністю, зручною для перевірки на належність значення категорії функцій, де функція - це особливий, що "визиваємий", об'єкт.

## Короткі описи типів даних
Усі типи даних JavaScript, крім об'єктів, є іммутабельними (значення не можуть бути модифіковані, лише перезаписані новим повним значенням). Наприклад, на відміну від C, де рядок можна посимвольно коригувати, JavaScript рядки перетворюються тільки повністю. Значення таких типів називаються "примітивними значеннями".

#### Числа:

Відповідно до стандарту ECMAScript, існує тільки один числовий тип, який являє собою [64-бітове число подвійної точності згідно зі стандартом IEEE 754](https://ru.wikipedia.org/wiki/Число_двойной_точности). Іншими словами, спеціального типу для цілих чисел JavaScript немає. Це означає, що при числових операціях ви можете отримати неточне (округлене) значення. Крім можливості представляти числа з плаваючою комою, є кілька символічних значень: ```+Infinity``` (позитивна нескінченність), ```-Infinity``` (негативна нескінченність), і ```NaN``` (не число).


#### Рядки:

JavaScript для представлення текстових даних служить тип String. Він є ланцюжком «елементів» 16-бітових беззнакових цілочисельних значень. Кожен такий елемент займає свою позицію у рядку. Перший елемент має індекс 0, наступний - 1, і так далі. Довжина рядка - це кількість елементів у ньому.
* Отримати частину вихідного рядка вибіркою окремих символів або застосуванням методу String.substr().
* Об'єднати два рядки в один, застосувавши оператор (+) або метод String.concat().


#### Булевий тип даних:
Булевий тип є логічною сутністю і має два значення: true (істина) і false (брехня).

#### Null:
Цей тип даних має лише одне значення: [null](https://developer.mozilla.org/ru/docs/Glossary/Null).


#### Undefined:
Змінна, якій не було присвоєно значення, матиме значення [undefined](https://developer.mozilla.org/ru/docs/Glossary/undefined).

#### Symbol:
Символи є нововведенням JavaScript починаючи з ECMAScript 2015. Символ – це унікальне та іммутабельне примітивне значення, яке може бути використане як ключ для властивості об'єкта. У деяких мовах програмування символи називаються атомами. Їх також можна порівняти з іменованими значеннями перерахування (enum) у мові C. Детальніше: [Symbol](https://developer.mozilla.org/ru/docs/conflicting/Web/JavaScript/Reference/Global_Objects/Symbol).


#### Велике число (BigInt):
BigInt є вбудованим об'єктом, який надає спосіб представлення цілих чисел, які більше 2 в 53 степені, що є найбільшим числом, яке JavaScript може надійно представити за допомогою [Number](https://developer.mozilla.org/ru/docs/Glossary/Number) примітиву.

<CodeEdit text={`
> let bigInt = 19241924124n;
> console.log(bigInt);
19241924124n
> console.log(typeof bigInt);
"bigint"
`}/>

#### Об'єкти
У комп'ютерній термінології об'єкт - це значення в пам'яті, на яке можна послатися за допомогою ідентифікатора. Дивитись про [Object](https://developer.mozilla.org/ru/docs/Web/JavaScript/Reference/Global_Objects/Object).

#### Дати
Для роботи з датами є вбудований глобальний об'єкт [Date](https://developer.mozilla.org/ru/docs/Web/JavaScript/Reference/Global_Objects/Date).
