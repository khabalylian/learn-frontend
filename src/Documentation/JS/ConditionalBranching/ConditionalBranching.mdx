# Умовні розгалуження: if else

Інструкція if виконує інструкцію, якщо зазначена умова виконується (істинно). Якщо умова не виконується (хибно), може бути виконана інша інструкція.

## Синтаксис

<CodeEdit text={`
if (умова){
   інструкція1
} else {
   інструкція2
}
`}/>

#### умова
Вираз, який є або істинним, або хибним.

#### інструкція1
Інструкція, що виконується у разі, якщо значення "умова" істинно ```(true)```. Може бути будь-якою інструкцією навіть вкладеним ```if```. Для угруповання кількох інструкцій використовується блок ```({...})```, коли жодної дії не потрібно, може використовуватися пуста інструкція.

#### інструкція2
Інструкція, що виконується у разі, якщо значення "умова" помилкове ```(false)```. Може бути будь-якою інструкцією, в тому числі вкладеною ```if```. Інструкції також можна групувати в блок.

## else if

Кілька команд ```if...else``` можуть бути вкладені для створення умови ```else if```. Зверніть увагу, що JavaScript немає ключового слова ```elseif`` (в одне слово).

<CodeEdit text={`
if (умова1){

   інструкція1 // Виконується якщо умова1 true

} else if (умова2) {

   інструкція2 // Виконується якщо умова1 false a умова2 true

} else if (умова3) {

   інструкція3 // Виконується якщо умова1 і умова2 false a умова3 true

} else {

   інструкція // Виконується якщо всі умови false

}
`}/>

## Умовний оператор ‘?’

Оператор представлений знаком питання ```?```. Іноді його називають “тернарним”, оскільки оператор має три операнди. Насправді це єдиний оператор у JavaScript, який має так багато операндів.

Синтаксис:

<CodeEdit text={`
let result = умова ? значення1 : значення2;
`}/>

Спочатку обчислюється ```умова```: якщо вона є правдивою, тоді повертається ```значення1```, інакше – ```значення2```.

Наприклад:

<CodeEdit text={`
let accessAllowed = (age > 18) ? true : false;
`}/>

З технічного погляду, ми можемо пропускати дужки навколо ```age > 18```. Оператор “знак питання” має низький пріоритет, тому він виконується після порівняння ```>```.

Але дужки роблять код більш читабельним, тому ми рекомендуємо їх використовувати.

<Alerts severity="info" variant="outlined">
	### Будь ласка, зверніть увагу:
	У наведеному вище прикладі можна уникнути використання оператора “знак питання”, оскільки само порівняння повертає true/false:

<CodeEdit text={`
// те ж саме
let accessAllowed = age > 18;
`}/>
</Alerts>

## Декілька ‘?’
Послідовність операторів знака питання ```?``` може повернути значення, яке залежить від більш ніж однієї умови.

Наприклад:

<CodeEdit text={`
let age = prompt('Вік?', 18);

let message = (age < 3) ? 'Привіт, крихітко!' :
  (age < 18) ? 'Вітаю!' :
  (age < 100) ? 'Моє шанування!' :
  'Який незвичайний вік!';

alert( message );
`}/>

Аналогічний код:

<CodeEdit text={`
let age = prompt('Вік?', 18);
let message = '';

if (age < 3){
   message = 'Привіт, крихітко!';
} else if (age < 18) {
   message = 'Вітаю!'
} else if (age < 100) {
   message = 'Моє шанування!'
} else {
    message = 'Який незвичайний вік!'
}

alert( message );
`}/>
